# Easy English Thai — Mobile App

Flutter application for pronunciation-first beginner English learning (Thai speakers).

## Stack

- **Flutter** 3.19+, Dart 3.3+
- **Firebase** — Auth, Firestore, Storage, Messaging
- **State management** — Riverpod 2 (code-gen)
- **Navigation** — GoRouter
- **Data models** — Freezed + json_serializable

## Architecture

Feature-first folder structure:

```
lib/
  core/
    firebase/     ← firebase_options.dart (generated by flutterfire)
    router/       ← GoRouter config + route constants
    theme/        ← AppTheme (Material 3)
    error/        ← error types and boundary widgets
  features/
    auth/         ← Firebase Auth (guest, email, upgrade)
    onboarding/   ← splash, goal selection, name, welcome
    home/         ← derived/home Firestore stream, today lesson, review due
    lesson/       ← lesson intro + step-by-step player
    review/       ← spaced repetition review queue
    dialogue/     ← guided/repeat/open dialogue sessions
    progress/     ← course completion, streak, weak sounds
    sounds/       ← sound detail, mouth diagram, drill
    settings/     ← notifications, voice speed, logout
  shared/
    widgets/      ← MainShell (bottom nav), reusable UI
    models/       ← shared enums and value types
    utils/        ← date helpers, formatters
```

## Setup

### 1. Install Flutter

Follow https://docs.flutter.dev/get-started/install

### 2. Configure Firebase

```bash
dart pub global activate flutterfire_cli
flutterfire configure
```

This replaces the placeholder `lib/core/firebase/firebase_options.dart`.

### 3. Install dependencies

```bash
flutter pub get
```

### 4. Run code generation

```bash
dart run build_runner build --delete-conflicting-outputs
```

### 5. Run the app

```bash
flutter run
```

## Firebase Emulator (local dev)

```bash
firebase emulators:start
```

Point the app to emulators by setting up `FirebaseEmulatorConfig` in your debug entry point.

## Backend decision

This app uses **Firebase** as the MVP backend:
- Firestore for all learner data and content
- Firebase Auth for authentication
- Firebase Storage for audio/image assets
- Firebase Messaging for review reminders
- Cloud Functions for derived document computation

See `../functions/` for the Cloud Functions implementation.
